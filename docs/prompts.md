# Claude Code 用プロンプト集
============================================================
このファイルの各プロンプトをそのまま Claude Code に貼り付けて実行できます。
スラッシュコマンド (`/wiki-ingest` 等) も利用可能です。
============================================================

---

## 📥 プロンプト: 素材の一括取り込み（Ingest）

```
CLAUDE.md を読んで、Ingest操作を実行してください。
raw/ 内でまだ wiki/sources/ に対応するファイルがないものを全て処理してください。
各ソースについて:
1. まず wiki/glossary.md を読んで既存用語を把握
2. wiki/sources/ に要約ファイルを作成（200-500語、重要引用を含む）
3. 主要な概念を抽出して wiki/concepts/ にページを作成/更新（既存用語と表記を統一）
4. 人物・組織を wiki/entities/ にページを作成/更新
5. wiki/glossary.md にエイリアス付きで新規用語を登録
6. wiki/index.md を更新
7. wiki/log.md に処理記録を追加
全ての [[wiki-links]] が正しくつながるようにしてください。
```

---

## 🔄 プロンプト: Wiki再構築（Compile）

```
CLAUDE.md を読んで、Compile操作を実行してください。
1. wiki/ 内の全ファイルをスキャン
2. 整合性チェック（index/glossary に漏れがないか修復）
3. wiki/index.md をゼロから再構築（concepts, entities, sources をカテゴリ別にリスト化）
4. wiki/glossary.md を再構築（エイリアス保持）
5. 全ての [[wiki-links]] が実在するファイルを指しているか検証
6. 重複する概念ページがあれば統合
7. 各ページのクロスリファレンスを更新
8. wiki/log.md に変更を記録
```

---

## ❓ プロンプト: 質問する（Query）

```
CLAUDE.md を読んで、Query操作を実行してください。
質問: [ここに質問を入力]

wiki/ 内の全ページを横断検索して、[[ソース名]] 付きで回答してください。
回答が実質的であれば wiki/outputs/ に保存してください。
関連する概念があればそれも提案してください。
```

---

## 🔍 プロンプト: 健康チェック（Lint）

```
CLAUDE.md を読んで、Lint操作を実行してください。
1. 壊れた [[wiki-links]] を全て検出
2. glossary.md の整合性チェック（未登録・孤立エントリ）
3. 表記揺れ候補の検出（類似した概念名のペア）
4. ソース間の矛盾を検出
5. 古い情報（1年以上前のソース）にフラグ
6. スタブページで拡張が必要なものを特定
7. カバレッジのギャップを報告（ファイルタイプ別）
8. 原文参照（Raw: パス）の存在確認
9. 可能な問題は自動修正
10. 全ての変更を wiki/log.md に記録
結果をサマリーレポートとして表示してください。
```

## 📎 プロンプト: 記事1本を即座に取り込み

```
以下のテキストを raw/articles/ に保存し、即座にIngestしてください:
- ファイル名は著者-年-短いタイトル.md のkebab-case形式で
- Source: [URL] を先頭に追加
- wiki/sources/ に要約を作成
- 新しい概念があれば wiki/concepts/ に作成
- wiki/index.md と wiki/log.md を更新

---
[ここに記事テキストを貼り付け]
```

---

## 🔀 プロンプト: Notionからの移行

```
raw/ に Notion のエクスポートファイル（Markdown）を配置しました。
CLAUDE.md を読んで、以下を実行してください:
1. 全ファイルをスキャンしてファイル名をkebab-caseに変換
2. Notion特有のプロパティやメタデータを整理
3. 通常のIngest操作として処理
4. 既存のwiki構造に統合
5. index と log を更新
```

---

## 📊 プロンプト: ナレッジベースの統計

```
現在のナレッジベースの状態を分析して報告してください:
- raw/ 内のファイル数（サブフォルダ別）
- wiki/ 内のファイル数（サブフォルダ別）
- 未処理のraw/ファイル数
- 壊れたリンク数
- 最も多くのリンクを持つ概念ページ（上位5件）
- 最後のIngest日時
- スタブページ数
```
