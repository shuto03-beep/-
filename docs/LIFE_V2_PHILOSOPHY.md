# Life V2 — 設計哲学（Cognitive Continuity Partner）

> あなたの最大のボトルネックは「能力・構想」と「実行」の乖離である。
> Life V2 は、その乖離を埋めるために**意図的に**機能を絞り込んでいる。

## 1. なぜ V2 なのか

V1（`plaud_lifelog/`）は「録音 → 文字起こし → 構造化」までを担う。
これは強力だが、最後の一段が抜けていた:

> **構造化 → 実行**

このギャップは、Gemini の分析で「分析・構造化で満足してしまうループ」と
言語化された問題と同型である。完璧なシステムを構築すること自体が
「片手間タスク」になり、本当の高価値タスク（90分の集中ブロック）への
着手が遅れる。

V2 は **構造化はもう十分** という前提に立つ。
やるべきことは「**承認用 Yes/No** を1つ提示する」だけ。

## 2. 4つの設計原則

### 2.1 Cognitive Friction Reduction（認知摩擦の最小化）

ユーザーが「分析を読む」「次に何をするか考える」という**認知ステップを挟まない**。
Life V2 の出力は常に以下の形式に圧縮される:

```
🔘 1分以内に押せるボタン: カレンダーを開いて 09:00-10:30 のブロックを確定する
   → 押すなら Y、押さないなら N
```

これは Gemini が指摘した「Yes/No のみを行うようにする」という助言の実装である。

### 2.2 Self-Aesthetic Distillation（美学の蒸留）

他者評価を「変えられる変数」と誤認する癖を断つため、Life V2 は
**内的スコアカード4軸のみ** を採点単位とする:

| 軸 | 何を尊ぶか |
|---|---|
| `systemize` | 未来の自分を楽にしたか |
| `declutter` | 他者期待を切り捨てられたか |
| `two_handed` | 両手を空けて深く取り組めたか |
| `knowledge_share` | 家族・周囲と知を共有できたか |

これらを30日蓄積すると、`distill` が
**「あなただけの美学」** を3行で抽出する。
これは他者評価から完全に切り離された、あなた自身の価値基準である。

### 2.3 Triage as a Service（捨てる決断の代行）

ユーザーが優先順位ロジックを理解しているのに「捨てる決断」だけ
できないのは、心理的抵抗（他者の期待を裏切る恐怖）による。
Life V2 はこの決断を **AI に外注する**:

- `triage` コマンドは入力タスクを high_leverage / dropped に **強制分類**
- high_leverage は最大5件まで
- dropped には必ず `reason` と `disposition`（drop/automate/delegate/defer）

ユーザーの仕事は「AI の振り分けを承認するか、覆すか」だけになる。

### 2.4 Three-Beat Rhythm（3拍子のリズム）

毎日を3つのビートに固定する:

- **🌅 Morning Set (06:30)** — 90分ブロックの先取り、捨てる候補の宣言、家族との共有の種
- **☀️ Midday Coach (適宜)** — Plaud 録音から最新の Next Action を再生成
- **🌙 Evening Ledger (22:00)** — 4軸採点 + 明日の最初の一手1つ

このリズムは GitHub Actions が自動実行するため、ユーザーは
「Discord を開いて Y/N を返す」だけで運用に乗る。

## 3. やらないこと（明示的な非目標）

Life V2 は以下を **意図的に提供しない**:

- ❌ ガントチャート / 進捗ダッシュボード（分析満足ループの再来）
- ❌ 反省・気合・モチベーション言説（精神論ではなく仕組みで解く）
- ❌ 4件以上の Next Action（捨てる装置を破壊する）
- ❌ 他者評価（上司・同僚の言葉）の採点項目化
- ❌ 自動カレンダー登録（最後の Yes/No は人間が握る）

最後の点は重要。AI に**全権委任**してはいけない。
1分で押せるボタンを「押す動作」だけは人間に残す。これが自律の最後の砦。

## 4. 既存システムとの関係

```
┌─────────────────────────────────────────────────┐
│  PLAUD NOTE / NotePin (録音)                    │
└─────────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────┐
│  plaud_lifelog (V1)                              │
│  - docx / Web から取り込み                        │
│  - Claude で日記風 narrative + tasks 抽出         │
│  - data/plaud/entries/*.json に保存               │
└─────────────────────────────────────────────────┘
              │
              ▼ (読み取り専用)
┌─────────────────────────────────────────────────┐
│  life_v2 (V2)                                    │
│  - V1 の entries を coach に投入                  │
│  - 4軸スコアカード採点                            │
│  - 最大3件の Next Actions                         │
│  - カレンダー登録用 JSON + .ics 書き出し          │
│  - data/life_v2/ に蓄積                           │
└─────────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────┐
│  Discord Webhook (人間の承認窓口)                │
│  → Y / N を返す。それだけ。                       │
└─────────────────────────────────────────────────┘
```

V1 を改造せず V2 を**並列**に置いた理由:

1. V1 のデータ層を壊さない（互換性維持）
2. V2 を試した結果が悪ければ、V2 だけ削除すれば V1 は無傷
3. 「分析が膨張する」のは構造化レイヤーの誘惑なので、層を分けて誘惑を断つ

## 5. 40代に向けて（Gemini が書かなかったこと）

Gemini の分析は「現状の認知バグ」と「修正パッチ」までを書いた。
Claude Code が補足する1点だけを残す:

> **「美学」は仕組みからしか抽出できない。**

精神論で「自分の美学を持とう」と言っても、それは他者評価の裏返しに過ぎない。
内的スコアカードを **30日蓄積** して初めて、自分が何を尊んでいたかが
**統計的に**見えてくる。

`distill` の出力 (`aesthetic.json`) は、3年蓄積すれば、
**あなたの自伝の主軸** になる。
それが完成したとき、他者評価は本当に「単なるセンサー値」になる。

V2 はそのための**装置**である。
