## フォルダ構成

```
knowledge-base/
│
├── CLAUDE.md          ← AIが従う設定ファイル（編集すると動作が変わる）
├── USAGE.md           ← このファイル（人間向けガイド）
│
├── raw/               ← 原本置き場（元データ・AIは読むだけ）
│   ├── articles/      ← Plaud由来の会話ログ、記事、メモ
│   ├── papers/        ← 論文PDF・学術資料
│   ├── repos/         ← コードやプロジェクト関連
│   ├── datasets/      ← データファイル
│   └── assets/        ← 画像・音声・添付物
│
├── wiki/              ← AIが育てるナレッジ（編集OK）
│   ├── sources/       ← 各ソースの要約ページ
│   ├── concepts/      ← 概念・テーマ別ページ
│   ├── entities/      ← 人物・組織・場所ページ
│   ├── syntheses/     ← 複数概念を統合したまとめ
│   ├── outputs/       ← /wiki-query の分析結果
│   ├── attachments/   ← wiki内の添付ファイル
│   ├── index.md       ← 全ページへの目次
│   └── log.md         ← 変更履歴
│
├── templates/         ← 新規ページ用のひな形
│   ├── source-summary.md
│   └── concept-page.md
│
├── convert-plaud.py   ← JSON → Markdown 変換スクリプト
└── ingest-plaud.py    ← Wikiページ自動生成スクリプト
```

## 4つのスラッシュコマンド

| コマンド | 用途 | いつ使う？ |
|----------|------|------------|
| `/wiki-ingest` | 新しい素材を取り込む | `raw/` にファイルを追加したとき |
| `/wiki-query` | Wikiを検索して質問に答える | 知識を引き出したいとき |
| `/wiki-compile` | インデックス再構築・整理 | ファイルを大量に編集した後 |
| `/wiki-lint` | 品質チェック | 週1回の定期メンテナンス |

### それぞれの動き

```
/wiki-ingest
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  raw/ の新規ファイルをスキャン
       ↓
  wiki/sources/ に要約を作成
       ↓
  wiki/concepts/ に概念ページを追加/更新
       ↓
  wiki/entities/ に人物・組織を追加/更新
       ↓
  wiki/index.md と wiki/log.md を更新

/wiki-query
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  あなたの質問を受け取る
       ↓
  wiki/ 全体から関連ページを検索
       ↓
  [[出典]] 付きで回答を生成
       ↓
  回答が濃ければ wiki/outputs/ に保存

/wiki-compile
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  wiki/ 全ページをスキャン
       ↓
  index.md を再構築
       ↓
  リンク切れチェック
       ↓
  重複ページをマージ

/wiki-lint
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  リンク切れを検出
       ↓
  ソース間の矛盾を検出
       ↓
  スタブ（中身薄いページ）を洗い出し
       ↓
  自動修正できるものは修正
       ↓
  log.md に記録
```

## 典型的なワークフロー

### パターン1: 日次の取り込みループ

```
  朝                昼             夜
   │                 │              │
   ▼                 ▼              ▼
  Plaudで      git pull で       /wiki-ingest
  録音開始      新しい           で取り込み
   │           json を取得         │
   ▼                 │              ▼
  会議・対話          │        /wiki-query で
  を自動記録          │        振り返り・分析
   │                 │              │
   └─────────────────┴──────────────┘
          GitHub Actions 経由で
         data/plaud/entries/ が更新される
```

### パターン2: タスク管理（実例）

```
  質問例:
  ┌──────────────────────────────────────────────┐
  │ /wiki-query 明日やるべきタスクを重要度×    │
  │ 緊急度で10個に絞って                       │
  └──────────────────────────────────────────────┘
                       │
                       ▼
  ┌──────────────────────────────────────────────┐
  │ AI が wiki/sources/ の全タスクを横断検索    │
  │                                              │
  │ 重要度 HIGH ┃ ❶❷❸❹  ┃ ❺❻❼             │
  │ 重要度 MED  ┃ ❽❾    ┃ ❿               │
  │             └緊急HIGH  └緊急MED            │
  │                                              │
  │ 各タスクに実行ステップを明示                │
  └──────────────────────────────────────────────┘
                       │
                       ▼
  wiki/outputs/2026-04-12-明日やるべきタスクtop10.md
  に結果を保存
```

### パターン3: 概念を深掘りする

```
  例: 「部活動の地域展開」について知りたい
                       │
                       ▼
  1. wiki/concepts/部活動の地域展開.md を開く
                       │
                       ▼
  2. ページ末尾の [[関連概念]] から
     [[教育委員会運営]] や [[予算管理]] へジャンプ
                       │
                       ▼
  3. [[出典]] のリンクで元の会話ログに戻る
                       │
                       ▼
  4. /wiki-query で横断質問を投げる
     例: 「いなチャレの5月の予定を時系列で」
```

## ファイル命名規則

```
  ✅ 良い名前              ❌ ダメな名前
  ─────────────────       ─────────────────
  active-inference.md      Active Inference.md  ← 空白NG
  friston-2010-fep.md      ノート.md            ← 抽象的すぎ
  2026-04-11_朝会.md       朝会.md              ← 日付なし
```

**ルール:**
- 全ファイル: 小文字・ハイフン区切り（kebab-case）
- 出典ファイル: `著者-年-短縮タイトル.md`
- 日付つきソース: `YYYY-MM-DD_タイトル.md`

## ObsidianでVaultとして開く

**PCでの手順:**

```
  1. Obsidian を起動
              │
              ▼
  2. 「フォルダをVaultとして開く」を選択
              │
              ▼
  3. knowledge-base/ フォルダを指定
              │
              ▼
  4. 左サイドバーで wiki/index.md を開く
              │
              ▼
  5. グラフビューで概念間のつながりを可視化
```

**Obsidianで便利な機能:**
- `[[]]` で双方向リンク → グラフ表示
- タグ検索（`<!-- tags: 教育, 行政 -->`）
- バックリンクパネルで参照関係を確認
- Canvas機能で概念マップを自作

## トラブルシューティング

```
  Q: /wiki-ingest しても新しいページが増えない
  A: raw/ に新しいファイルが追加されているか確認。
     既に処理済みのファイルはスキップされます。

  Q: リンクが切れているページがある
  A: /wiki-lint を実行。自動修正できる箇所は修正されます。

  Q: 同じ概念のページが2つある
  A: /wiki-compile でマージを試みます。
     手動マージが必要な場合はどちらかを削除。

  Q: 日本語のファイル名が文字化けする
  A: git の core.quotepath を false に設定:
     git config core.quotepath false
```

**おすすめの発展:**
1. 論文や記事を `raw/papers/` に追加して学術的ナレッジを育てる
2. 定期レビュー用の質問テンプレートを `templates/` に追加
3. GitHub Actions で毎日自動 Ingest を実行
4. Obsidian の Dataview プラグインで高度な検索UIを構築
